# 职能分工总览

## 摘要

- 本页用于管理 `Game_WaJue` 的开发资料目录。
- 当前按五类入口组织：策划、程序、美术、质量管理 / QA，以及跨职能的公共。
- 资料和资产必须优先放入对应职能文件夹；如果一份资料会影响多个职能，在主责文件夹保存，并在相关页面中互相链接。
- 外部文章、视频、参考图等仍先登记到 `原始资料/来源登记/`；下载后的原始附件按主责职能放入 `原始资料/附件/职能分工/对应职能/`。

## Unity 工程映射

- 外部 Unity/Tuanjie 工程：`E:\GitHub\Unity_Pro\WeChat_Game`。
- Unity 内的职能资源根目录采用“拼音 + 大驼峰”命名，只保留三个入口：
  - `Assets/CeHua/`：策划内容。
  - `Assets/ChengXu/`：程序内容。
  - `Assets/MeiShu/`：美术内容。
- 后续不同职能的 Unity 内容都存入对应入口；若继续细分子目录，也沿用拼音 + 大驼峰命名。
- 质量管理 / QA 与公共内容主要是文档、证据和记录，默认存放在本知识库；如确实产生 Unity 资源，由项目管理判断主责后放入 `Assets/CeHua/`、`Assets/ChengXu/` 或 `Assets/MeiShu/`。
- 本页下方“子目录一览”用于知识库资料管理，不再代表 Unity 内需要创建同名中文目录。

## 使用规则

1. **先分主责**：谁负责产出或维护，就放到谁的文件夹。
2. **再做链接**：涉及其他职能时，不复制多份，使用 Obsidian 链接互相引用。
3. **需求和决策要沉淀**：可复用的分析、取舍、评审结论，后续可整理到 `知识库/问题/`。
4. **QA 独立存放**：验收清单、测试用例、缺陷记录、回归验证和质量风险放入 [[知识库/项目/职能分工/质量管理/说明]]。
5. **跨职能放公共**：里程碑、跨职能任务、会议 / 评审记录、共同决策放入 [[知识库/项目/职能分工/公共/说明]]。
6. **附件也分职能**：参考图、截图、录屏、导出包、日志、配置样例等原始附件，按主责放入 `原始资料/附件/职能分工/策划/`、`原始资料/附件/职能分工/程序/`、`原始资料/附件/职能分工/美术/`、`原始资料/附件/职能分工/质量管理/` 或 `原始资料/附件/职能分工/公共/`。
7. **素材来源要可追溯**：外部参考、竞品截图、文章和视频优先登记到 `原始资料/来源登记/`。
8. **少建空页**：先用文件夹和说明页定边界，等有真实内容后再创建具体页面。
