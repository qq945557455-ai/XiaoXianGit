# 质量管理 / QA 说明

## 摘要

- 本页定义 `质量管理 / QA` 环节资料的收纳边界。
- QA 是独立职能：验收清单、测试用例、缺陷记录、回归验证和质量风险，都放在本目录下。
- QA 产生的截图、录屏、日志、测试证据等原始附件，放入 `原始资料/附件/职能分工/质量管理/`。
- 如果 QA 事项同时影响策划、美术、程序和排期，结论或会议记录放入 [[知识库/项目/职能分工/公共/说明]]，本页只保留 QA 专项材料和链接。

## 适合放这里

- **验收清单**：需求验收标准、功能验收表、版本验收门禁。
- **测试用例**：玩法测试、UI 测试、性能测试、兼容性测试、异常流程测试。
- **缺陷记录**：Bug 复现步骤、期望结果、实际结果、严重程度、责任职能、修复状态。
- **回归验证**：修复后验证记录、回归范围、是否通过、遗留风险。
- **质量风险**：体验风险、稳定性风险、性能风险、上线阻塞、需要管理者确认的问题。

## 子文件夹

- `验收清单/`
- `测试用例/`
- `缺陷记录/`
- `回归验证/`
- `质量风险/`

## 存放规则

1. 单个需求的 QA 材料，优先放入本目录对应子文件夹。
2. 需求卡、排期、跨职能评审和最终决策放入 [[知识库/项目/职能分工/公共/说明]]。
3. Bug 如果需要程序、美术或策划修复，缺陷记录仍在 QA，责任职能页面只链接对应记录。
4. 截图、录屏、日志等原始证据放在 `原始资料/附件/职能分工/质量管理/`，不要散落在其他附件目录。
