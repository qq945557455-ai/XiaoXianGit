# 游戏规划

## 摘要

- 本页作为 `Game_WaJue` 的开发规划入口。
- 当前开发资料按职能分为：策划、程序、美术、质量管理 / QA，并额外建立公共文件夹承接跨职能事项。
- 详细目录与使用规则见 [[知识库/项目/职能分工/总览]]。

## 职能目录

| 职能 | 文件夹入口 | 主要内容 |
| --- | --- | --- |
| 策划 | [[知识库/项目/职能分工/策划/说明]] | 核心玩法、系统规则、关卡内容、数值经济、剧情世界观、版本需求 |
| 程序 | [[知识库/项目/职能分工/程序/说明]] | 工程架构、玩法实现、UI 与交互、数据存档、工具链、性能调试 |
| 美术 | [[知识库/项目/职能分工/美术/说明]] | 美术风格、角色、场景、UI 图标、动画特效、素材导出规范 |
| 质量管理 / QA | [[知识库/项目/职能分工/质量管理/说明]] | 验收清单、测试用例、缺陷记录、回归验证、质量风险 |
| 公共 | [[知识库/项目/职能分工/公共/说明]] | 里程碑、任务清单、评审记录、跨职能决策 |

## 使用建议

- 新想法先写到对应职能目录；如果影响多个职能或还没确定归属，先放到公共文件夹或当前页的待办区。
- 有外部资料时，先登记到 `原始资料/来源登记/`，再整理为 `知识库/来源摘要/` 或具体项目页。
- 形成可复用决策时，沉淀到 `知识库/问题/`，避免只留在聊天记录里。
