# 需求看板

## 摘要

本看板用于快速查看管理者对各职能下达的需求、完成进度、评分和相关跳转链接。  
这里只放需求和产出结果大纲，不放流程图、制度长文或详细讨论过程。

## 字段说明

- **职能**：策划 / 美术 / 程序 / 质量管理 / 公共。
- **完成进度**：建议使用 `0% / 25% / 50% / 75% / 90% / 100%`。
- **评分**：只使用 `低 / 中 / 高`。
  - `低`：结果未达到当前要求，需要明显返工。
  - `中`：结果基本可用，但仍需调整或补充。
  - `高`：结果达到当前要求，可以继续推进或作为参考沉淀。
- **相关跳转**：放需求、产出、说明页或附件索引的 Obsidian 链接。

## 新增需求记录模板

新增需求时，复制一行到“总需求看板”，再按职能复制到对应分区：

```markdown
| REQ-YYYY-MMDD-编号 | 需求名称 | 职能 | 需求内容 | 产出结果大纲 | 待澄清 / 待拆分 / 进行中 / 待验收 / 已完成 / 阻塞 | 0% | 低 / 中 / 高 | 相关笔记链接 | 下一步 |
```
