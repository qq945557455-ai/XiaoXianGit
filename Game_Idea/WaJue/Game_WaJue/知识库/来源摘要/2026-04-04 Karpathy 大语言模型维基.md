# Karpathy — 大语言模型维基

## 摘要

这篇文章提出一种不同于传统检索增强生成的个人知识库模式：大语言模型不只是临时检索原始文档，而是持续维护一个由 Markdown 页面组成的维基。每次新增来源或提出重要问题时，大语言模型都把信息整合到已有页面、补充交叉链接、标记矛盾、更新索引和日志，让知识库随时间复利。

核心架构分为三层：原始资料、知识层、规则层。原始资料是事实来源，知识层是大语言模型维护的综合层，规则层是约束大语言模型行为的规范文件。核心操作包括摄入、查询、体检：摄入资料、回答并沉淀问题、定期体检知识库。

## 关键方法论

### 1. 不把知识库停留在“可检索文件夹”

传统检索增强生成往往在每次提问时重新检索、重新拼装答案。大语言模型维基的目标是把已读资料编译成稳定、可更新、可链接的知识资产。

### 2. 人和大语言模型分工

- 人：选择资料、决定研究方向、提出好问题、判断哪些洞见重要。
- 大语言模型：摘要、归档、交叉链接、更新旧页面、记录日志、发现缺口。

### 3. 三层架构

- `原始资料/`：不可随意改写的来源层。
- `知识库/`：可演化的综合知识层。
- `AGENTS.md`：告诉大语言模型如何维护知识库的规则层。

### 4. 三类操作

- **摄入**：读新资料，写摘要，更新相关页面和索引。
- **查询**：从已有知识层回答问题，并把有价值回答沉淀回知识库。
- **体检**：检查矛盾、过时内容、孤立页、缺失概念和研究缺口。

### 5. 索引与日志

- `索引.md` 面向内容，是导航和检索入口。
- `日志.md` 面向时间，是知识库演化记录。
