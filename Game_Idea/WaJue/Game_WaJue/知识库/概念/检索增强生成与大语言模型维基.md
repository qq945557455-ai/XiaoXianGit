# 检索增强生成与大语言模型维基

## 摘要

检索增强生成和大语言模型维基都能帮助大语言模型使用外部资料，但侧重点不同。检索增强生成更像“查询时临时检索资料”，大语言模型维基更像“提前把资料整理成可持续维护的知识层”。两者并不冲突：小中型知识库可先依赖索引和 Markdown 搜索；规模变大后可以在知识层之上再加搜索或向量检索。

## 对比

| 维度 | 检索增强生成 | 大语言模型维基 |
|---|---|---|
| 主要动作 | 查询时检索片段 | 摄入时整理和整合 |
| 记忆方式 | 原始文档 + 索引 | 原始文档 + 互联维基 |
| 知识积累 | 每次回答可能重新拼装 | 每次摄入和问答都沉淀 |
| 矛盾处理 | 常在回答时临时发现 | 可在维护中显式记录 |
| 人的体验 | 上传文件后提问 | 像浏览和维护一个活的维基 |

## 何时用哪种

- 资料少、需要快速问答：检索增强生成足够。
- 需要长期研究、综合和演化：大语言模型维基更合适。
- 页面数量增多、检索成本上升：在大语言模型维基上叠加本地搜索或向量检索。
