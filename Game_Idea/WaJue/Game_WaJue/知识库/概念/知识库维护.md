# 知识库维护

## 摘要

本知识库按“大语言模型维基”方式维护：原始资料作为事实来源，知识层负责摘要、整合、链接和演化，规范文件约束目录、模板和工作流。维护的重点不是单次回答，而是把每次摄入、提问和设计决策沉淀为可复用资产。

## 关键点

- 新资料先登记来源，再写来源摘要，并更新相关概念、实体或项目页。
- 重要问答要沉淀到问题页或相关主题页，避免只留在聊天记录里。
- 索引 [[知识库/索引]] 和日志 [[知识库/日志]] 是导航与追踪入口，每次重要改动后都应同步更新。
- 定期做 [[知识库/概念/知识库健康检查]]，清理命名漂移、缺失链接、重复概念和过时结论。
